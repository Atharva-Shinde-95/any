1️⃣ What “State Memory” Means in LangGraph (Important)
In LangGraph, there is one shared state object that:

Is passed into every node
Is returned out of every node
Is the only communication channel between nodes

So yes:
✅ Every agent reads from state
✅ Every agent writes to state
✅ No agent talks to another agent directly
This is by design, not accidental.

2️⃣ What Kind of Data Flows Through State
The state contains structured fields, for example:
Plain Textstate = {  raw_data,  data_profile,  canonical_model,  inventory_allocation,  batch_plan,  routing_plan,  schedule,  validation_result,  final_output}Show more lines
✅ Agents append or update specific fields
✅ Agents must not overwrite unrelated fields
✅ Validation agent decides routing based on state content

3️⃣ What Does NOT Flow Through State Automatically
This is critical.
❌ Not automatic internal thoughts

LLM chain‑of‑thought
Agent reasoning scratchpads
Tool internal logs

These remain inside the agent invocation, not in state.

❌ Not “agent memory” unless you explicitly store it
Conversation history, summaries, or reasoning only appear in state if you explicitly write them there.
