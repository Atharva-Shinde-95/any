User uploads the data 
 
Then Data Audit Node, where python scans the full file and checks for empty sheets, missing values, duplicate rows. 
Then an LLM looks at the sheets and figures out which ones are actually related to each other (like orders linking to timelines, products linking to machines). 
If anything is broken here, pipeline stops and tells you exactly why.
 
If everything is fine and satisfied then Schema Mapping Starts
Schema Reference (optional) Client can describe their columns in plain English, like "this column is the order quantity, this one links to machines." The LLM reads that and uses it as trusted guidance for the next step.  
 
Then Schema Mapping Agent
This is the main AI agent. It explores the workbook autonomously using tools, figures out which column maps to which scheduling concept (order ID, machine capacity, deadlines etc.), verifies the foreign key relationships between sheets, and submits the full mapping. 
 
Then Human Review comes
The mapped concepts show up in a table in the UI. The person can edit any row, approve the mapping, or reject and redo. Only after human approval does the pipeline continue. 
 
after this is done the problem understanding comes where
LLM reads the client's scheduling requirement and extracts the problem type, objective, CP-SAT variables, and constraints.
Final check confirms all required data is present.
